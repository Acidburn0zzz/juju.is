What is Juju?

Built to manage complex software architectures wherever they’re run, Juju simplifies deployment and post-install operations like upgrades, updates and configuration management.

Use it to manage applications and services on Kubernetes, VMs, bare metal or any hybrid cloud configuration.

Getting started

The first part of the getting started section introduces you to Juju by first experimenting with a local deployment on an Ubuntu VM. The second part outlines the process for deploying to a public cloud.

  • Launch an instance (by default you get the current Ubuntu LTS)

    Run commands in that instance, try running bash (logout or ctrl-d to quit)

    Pass a cloud-init metadata file to an instance on launch see using cloud-init with multipass for more details

    See your instances

    Stop and start instances

    Clean up what you don’t need

    Find alternate images to launch with multipass

    Get help

    Now don’t forget you still have ‘foo’ running :)

JAAS is Juju as a service.

Hosted by Canonical, the makers of Ubuntu, JAAS provides managed Juju infrastructure to operate your applications. It’s the fastest way to operate your workloads on supported clouds with the fewest instances required.

Build, configure, manage and deploy your applications to the public cloud directly from your browser or the command line.
